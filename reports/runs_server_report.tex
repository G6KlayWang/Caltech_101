\begin{abstract}
This report summarizes training and evaluation results for Caltech-101 classification over 102 classes (including \texttt{BACKGROUND\_Google}). We compare two classical baselines (HOG + Linear SVM, HOG + Engineered Features + LightGBM) against transfer-learning models (ResNet50, EfficientNet-B0, ViT-B/16), then analyze ablations on image size, augmentation, and optimizer. On the main test runs, ViT-B/16 achieves the best Top-1 accuracy (0.9461) and Macro-F1 (0.9272), followed by ResNet50 (0.9402, 0.9183) and EfficientNet-B0 (0.9213, 0.8955). Among classical methods, LightGBM improves over SVM (Top-1 0.5437 vs 0.5036), but both remain substantially below deep transfer-learning performance. Ablations show strong dependence on image resolution for CNNs, limited gains from the tested augmentation setup, and a large optimizer sensitivity for ResNet50 (Adam $\gg$ SGD in this setup).
\end{abstract}

\section{Introduction}
Caltech-101 is an important benchmark for multi-class object recognition, and it has been widely used to evaluate both classical and modern visual representation learning methods \cite{feifei2004caltech101}. Traditional pipelines often combine hand-crafted Histogram of Oriented Gradients (HOG) descriptors \cite{dalal2005hog} with Support Vector Machine (SVM) classifiers \cite{cortes1995svm}, while recent progress is dominated by deep architectures such as ResNet \cite{he2016resnet}, EfficientNet \cite{tan2019efficientnet}, and Vision Transformer (ViT) \cite{dosovitskiy2021vit}.

In this work, we evaluate Caltech-101 classification under a unified experimental framework using all 102 classes, including \texttt{BACKGROUND\_Google}, with a stratified split protocol (train/val/test = 6400/1372/1372 images, i.e., 70.0\%/15.0\%/15.0\%). We implement a classical baseline (HOG + Linear SVM) and multiple transfer-learning deep models (ResNet50, EfficientNet-B0, and ViT-B/16), report Top-1 accuracy, Top-5 accuracy, Macro-F1, and Weighted-F1, and run ablation studies on image resolution, augmentation policy, and optimizer choice.

The purpose of this report is to provide a rigorous empirical comparison between classical machine learning models and deep learning models under the same split protocol, and to clearly characterize the performance differences between these two model families on Caltech-101.

\section{Dataset}
All experiments use Caltech-101 \cite{feifei2004caltech101}, with 9,144 images across 102 classes. We construct data splits with a two-stage stratified procedure over class labels: first, 70\% training and 30\% temporary data; second, the temporary set is split evenly into validation and test sets, yielding 6,400/1,372/1,372 images (70.0\%/15.0\%/15.0\%). This preserves class proportions across splits and avoids sample overlap.

We intentionally keep \texttt{BACKGROUND\_Google} as a valid class, rather than discarding it, to better reflect realistic recognition conditions where images may not correspond to a clean foreground object category. Including this class makes the task more challenging and more representative, and it allows us to evaluate whether models can separate true object categories from heterogeneous background-like content.

Preprocessing is model-specific but standardized within each family. For the classical pipeline, each image is first converted to RGB to standardize mixed source formats into a consistent 3-channel input, then resized to 128$\times$128 to keep feature dimensionality fixed, scaled to $[0,1]$ for numerical stability, converted to grayscale so HOG focuses on intensity-gradient structure, and finally encoded with HOG features (9 orientations, 8$\times$8 pixels per cell, 2$\times$2 cells per block). For deep models, ResNet50 and EfficientNet-B0 are trained at 128$\times$128, while ViT-B/16 is run at 224$\times$224 to satisfy architecture input requirements; all deep inputs are normalized with ImageNet mean/std to match pretrained feature statistics, and training uses either deterministic resize without augmentation or RandomResizedCrop + horizontal flip + color jitter with augmentation for regularization, while validation/test always use deterministic resize and normalization for consistent evaluation.

\section{Methods}
\subsection{HOG + Linear SVM}
As a classical baseline, we employ a linear Support Vector Machine (SVM) on HOG representations. Model selection is performed on the validation split by maximizing Macro-F1 over the regularization parameter $C$, and the selected model is evaluated on the held-out test set.

\subsection{HOG + Engineered Features + LightGBM}
To provide a stronger non-linear classical comparator, we concatenate HOG features with nine engineered image statistics (width, height, aspect ratio, channel-wise RGB means, and channel-wise RGB standard deviations) and train a multiclass LightGBM classifier. Hyperparameters (\texttt{num\_leaves}, \texttt{learning\_rate}, \texttt{n\_estimators}) are selected using validation Macro-F1.

\subsection{Transfer-Learning Deep Models}
We fine-tune three ImageNet-pretrained backbones and adapt each architecture to 102-way classification by replacing the final prediction head.

\subsubsection{ResNet50}
ResNet50 is a residual convolutional architecture in which skip connections enable stable optimization of deep networks. We replace the final fully connected layer with a 102-class classifier and fine-tune the model with cross-entropy, selecting the checkpoint that achieves the best validation Macro-F1.

\subsubsection{EfficientNet-B0}
EfficientNet-B0 is a compact convolutional model that balances depth, width, and resolution through compound scaling. We replace its classifier layer with a 102-class head and fine-tune it under the same training/selection criterion, retaining the best validation Macro-F1 checkpoint.

\subsubsection{ViT-B/16}
ViT-B/16 is a transformer-based vision architecture that models global image context via self-attention over patch tokens. We replace the transformer head with a 102-class classifier and fine-tune the model with cross-entropy, using the best validation Macro-F1 checkpoint for test evaluation.

\subsection{Training Protocol}
For all deep models, training lasts 12 epochs with a two-stage freeze--unfreeze strategy: the backbone is frozen for the first 2 epochs and then unfrozen for full fine-tuning. ResNet50 and EfficientNet-B0 use batch size 32, initial learning rate $3\times10^{-4}$, and post-unfreeze learning rate $1\times10^{-4}$ at $128\times128$ resolution, whereas ViT-B/16 uses batch size 16, initial learning rate $1\times10^{-4}$, and post-unfreeze learning rate $5\times10^{-5}$ at $224\times224$; in all deep runs, Adam is the default optimizer with weight decay $1\times10^{-4}$.

For the classical models, HOG extraction uses size 128, 9 orientations, 8$\times$8 pixels per cell, and 2$\times$2 cells per block. The linear SVM is trained with \texttt{max\_iter}=1000 and validation tuning over $C$ (configured grid: [0.1]). The LightGBM baseline is intentionally configured for fast execution with a single setting (\texttt{num\_leaves}=31, \texttt{learning\_rate}=0.1, \texttt{n\_estimators}=100, \texttt{subsample}=0.8, \texttt{colsample\_bytree}=0.8), and model selection remains based on validation Macro-F1; GPU mode can be requested in the training script, with automatic CPU fallback if GPU acceleration is unavailable in the LightGBM build.

\section{Experiment Setup}
\subsection{Augmentation}
To isolate the effect of data augmentation, we evaluate two training pipelines for deep models while keeping the remaining settings fixed. The first pipeline applies only deterministic resizing to the target input resolution. The second pipeline applies stochastic augmentation during training (random resized crop, horizontal flip, and color jitter), which acts as regularization by increasing appearance variability. For both pipelines, validation and test images are processed with deterministic resizing and normalization to ensure a controlled and comparable evaluation protocol.

\subsection{Optimization}
Deep models are optimized with cross-entropy loss under a freeze--unfreeze transfer-learning schedule: the pretrained backbone is frozen in early epochs and then unfrozen for end-to-end fine-tuning. Adam is used as the default optimizer in the main comparison runs, with weight decay applied as regularization. To explicitly assess optimizer effects, we also run a controlled comparison in which ResNet50 is trained twice under identical image size and augmentation settings, once with Adam and once with SGD.

\subsection{Model Selection}
For deep models, checkpoint selection is based on validation Macro-F1, and the best checkpoint is used for final test evaluation. We prefer Macro-F1 over overall accuracy because the class distribution is imbalanced; accuracy can be dominated by frequent classes, whereas Macro-F1 gives equal weight to each class and therefore better reflects balanced recognition quality across all 102 categories.

\subsection{Ablation Protocol}
\begin{itemize}
    \item \textbf{A1 (Image Resolution):} ResNet50 and EfficientNet-B0 are evaluated at 64 and 128 input resolution, with all other settings fixed. ViT-B/16 remains at 224 due to architecture constraints.
    \item \textbf{A2 (Augmentation Effect):} each deep model is trained with and without stochastic augmentation at fixed resolution and optimizer settings.
    \item \textbf{A3 (Optimizer Effect):} ResNet50 is trained with Adam and SGD under matched settings to isolate optimizer impact.
\end{itemize}

\subsection{Per-Class Results}
To quantify class-wise behavior, we report per-class accuracy for each class $c$ as
\[
\mathrm{Acc}_c = \frac{\#\{\text{correct predictions in class } c\}}{N_c} = \frac{\mathrm{TP}_c}{N_c},
\]
where $N_c$ is the number of test samples in class $c$. We summarize these values by their arithmetic mean across classes:
\[
\overline{\mathrm{Acc}}_{\mathrm{class}} = \frac{1}{C}\sum_{c=1}^{C}\mathrm{Acc}_c,\quad C=102.
\]
This analysis is necessary because the dataset is class-imbalanced; aggregate accuracy can mask poor performance on low-frequency or visually difficult categories.

Average per-class accuracy across the 102 classes is:
\begin{itemize}
    \item Classical SVM: 0.2893
    \item Classical LightGBM: 0.3076
    \item EfficientNet-B0: 0.8977
    \item ResNet50: 0.9222
    \item ViT-B/16: 0.9350
\end{itemize}
Additionally, 37 classes are predicted perfectly (accuracy = 1.0) by all three deep models.

\section{Ablation Study}
We conduct a targeted ablation study to isolate the contribution of three implementation factors in the deep-learning pipeline: input resolution, augmentation policy, and optimizer choice. Each ablation changes one factor while holding the others fixed, so that performance differences can be interpreted causally rather than as the result of multiple simultaneous configuration changes.

\centering

\midrule

Increasing resolution from 64 to 128 substantially improves both Top-1 and Macro-F1 for CNN backbones, with the largest gain on EfficientNet-B0. The larger Macro-F1 gains (especially +0.1528 for EfficientNet-B0) indicate that higher resolution benefits not only overall accuracy but also balanced performance across classes. ViT-B/16 remains unchanged because its implementation is constrained to 224 input resolution in both settings.

\section{Observation and Discussion}
\subsection{Deep Models vs Classical Baselines}
Deep transfer-learning models substantially outperform the classical baselines, with ViT-B/16 achieving the best overall performance (Top-1 0.9461, Macro-F1 0.9272). This gap indicates that pretrained deep representations are the most reliable option for this 102-class setting, especially when balanced class-level performance is a priority.

\subsection{SVM vs LightGBM Baseline Behavior}
LightGBM improves Top-1 and Weighted-F1 over linear SVM, indicating better non-linear fitting capacity for global decision boundaries. However, its Macro-F1 remains low and close to SVM, implying that classical pipelines still struggle to generalize across rare and difficult categories.

\subsection{Implications from Resolution and Optimizer Ablations}
The ablations show that image resolution and optimizer selection are high-impact controls in this pipeline. In practical terms, moving from 64 to 128 strongly improves CNN performance, and Adam is clearly more suitable than SGD under the tested ResNet50 hyperparameters.

\subsection{Implications from Augmentation Ablation}
The tested augmentation policy yields consistent metric decreases across all three deep models. This suggests that augmentation should be treated as a tunable design variable rather than a default improvement, and its strength/composition should be re-optimized for this dataset and training budget.

\subsection{Failure Modes}
A consistent failure pattern is concentrated on classes with small test support and/or high visual ambiguity. For example, classes such as \texttt{lotus}, \texttt{crocodile}, and \texttt{snoopy} remain among the hardest even for strong deep models, while several low-support classes reach near-zero accuracy under classical models. The confusion matrices also indicate that heterogeneous categories (including \texttt{BACKGROUND\_Google}) are more prone to cross-class confusion, which suggests sensitivity to cluttered scenes, intra-class variability, and limited sample diversity in minority classes.

\subsection{Limitations}
This study is limited by a single split protocol and a fixed seed in the reported main runs, so variance across random initializations and split realizations is not fully characterized. Hyperparameter exploration is also intentionally constrained for some settings (notably the fast LightGBM baseline), which prioritizes runtime over peak performance. In addition, ViT-B/16 input size is architecture-constrained in this codebase, and augmentation exploration is restricted to one policy family; broader augmentation and optimization sweeps may yield different conclusions. Finally, results are reported on a single dataset benchmark, so cross-dataset generalization is not evaluated.
